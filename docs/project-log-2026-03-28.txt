PROJECT CONCEPT (FOR TEAM REFERENCE)
================================================================================

We are building a web-based "Interactive Memory Machine" — a system that acts like a spatial memory analyst or therapist.

How it works:
  1. A user (subject) writes two texts:
     - A MEMORY: a recent spatial experience (waking, real)
     - A DREAM: the subconscious reconstruction of that same space

  2. The system analyzes the differences between memory and dream, identifying three types of "glitch":
     - OMISSION: elements present in memory but absent from dream
     - OBSCURATION: elements present in both but with reduced clarity in the dream
     - MISASSOCIATION: elements with incorrect spatial relationships in the dream

  3. The system generates:
     - Side-by-side images (memory visualization vs. dream visualization)
     - A 10-second video walkthrough of the distorted interior space
     - Full documentation of the analysis process (like therapist case notes)

The "glitch" is NOT a visual filter or aesthetic effect. It represents meaningful distortion — how the mind remembers, forgets, blurs, and reorganizes spatial experience.

TECH STACK & DECISIONS
================================================================================

Framework:       Next.js 15 (App Router) + TypeScript + Tailwind CSS
  - Why: Single framework handles both the frontend UI and backend API routes.
    No need to set up a separate server. Simple to deploy.

Image Generation: fal.ai — Flux Schnell (development) / Flux Pro (production)
  - Why: Best quality for architectural/spatial imagery.
    Cost: ~$0.03-0.05 per image.

Video Generation: fal.ai — Kling v2 Master (image-to-video)
  - Why: Good camera movement control, architectural scene coherence.
    Cost: ~$0.07/second (~$0.70 for a 10-second video).

Interpretive Agent: Claude API (Sonnet 4)
  - Why: Claude acts as the "analyst/therapist" — it reads both texts and produces
    structured analysis of how memory has shifted. This is the conceptual core.
    Cost: ~$0.01 per session (negligible).

Hosting: Vercel (free tier)
  - Why: One-click deploy from GitHub, zero DevOps needed.

Why fal.ai over alternatives:
  - Single unified API for BOTH image and video generation
  - One SDK, one billing account, one set of docs
  - Avoids juggling Replicate + RunwayML + Stability AI separately

Estimated cost per full session: ~$0.80-1.00
Estimated total project budget:  ~$30-50

PIPELINE ARCHITECTURE
================================================================================

The pipeline follows our 6-step methodology from the presentation:

  USER INPUT                     API CALLS                       OUTPUT
  -----------                    ---------                       ------
  Step 1: Memory text --+
                        +---> Step 3-4: Claude API ---> Analysis JSON
  Step 1: Dream text  --+         |                     (analyst notes +
                                  |                      distortion categories)
                                  v
                        Step 2: fal.ai Flux ---> Memory image
                        Step 2: fal.ai Flux ---> Dream image
                                  |
                                  v
                        Step 5: Claude API ---> Spatial translation
                                  |              (video prompt with
                                  |               distortion instructions)
                                  v
                        Step 6: fal.ai Kling ---> 10-second video walkthrough


Claude AI is called TWICE per session:

  Call 1 — ANALYSIS (Steps 3-4):
    Input:  Memory text + Dream text
    Output: Structured JSON with distortions categorized as
            OMISSION / OBSCURATION / MISASSOCIATION,
            each with element name, interpretation, severity (1-5).
            Plus 2-3 paragraphs of prose "analyst notes."

  Call 2 — SPATIAL TRANSLATION (Step 5):
    Input:  The analysis JSON from Call 1
    Output: Detailed architectural image prompt (describing the distorted space)
            + video prompt (camera movement through the space)
            + style modifiers


================================================================================
CLAUDE AI SYSTEM PROMPTS (FULL TEXT)
================================================================================

--- ANALYSIS PROMPT ---

"You are a spatial memory analyst studying how architectural experiences
transform in subconscious recall. You analyze the differences between a
subject's waking spatial memory and their dream reconstruction of the
same space.

Your role is clinical and interpretive — like a therapist examining how
the mind distorts, omits, and reorganizes spatial experience during sleep.

Categorize every difference into exactly one of:
- OMISSION: Elements present in memory but absent from dream
- OBSCURATION: Elements present in both but with reduced clarity, blurred
  boundaries, or diminished detail in the dream
- MISASSOCIATION: Elements present in both but with incorrect spatial
  relationships, wrong contexts, or displaced connections"

Output format: JSON with subject_id, distortions array, overall_pattern,
and analyst_notes.


--- SPATIAL TRANSLATION PROMPT ---

"You are translating psychological spatial distortions into concrete
architectural visualization instructions. Given an analysis of memory
distortions, generate prompts for AI image and video generators that
will create a walkthrough of an interior space shaped by these distortions.

Map each distortion type to spatial conditions:
- OMISSION -> gaps, missing walls, incomplete rooms, void spaces
- OBSCURATION -> fog, blur, dissolving surfaces, uncertain boundaries
- MISASSOCIATION -> impossible adjacencies, objects in wrong rooms,
  scale distortions, spatial contradictions

The generated space should feel both familiar and deeply wrong — like a
place you almost recognize but that follows dream logic rather than
physical logic."

Output format: JSON with image_prompt, video_prompt, and style_modifiers.
